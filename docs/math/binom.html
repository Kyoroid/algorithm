二項係数・組み合わせ¶

https://drken1215.hatenablog.com/entry/2018/06/08/210000

二項係数¶

二項係数 (binomial coefficient) は、\((1+x)^n\) を展開した時に現れる係数のこと。
\(_n\mathrm{C}_k\) または \(\binom{n}{k}\) と表記する。
\[\begin{split}\begin{eqnarray} (1+x)^0 &=& 1 \\ (1+x)^1 &=& 1 + x \\ (1+x)^2 &=& 1 + 2x + x^2 \\ (1+x)^3 &=& 1 + 3x + 3x^2 + x^3 \\ &\vdots& \\ (1+x)^n &=& _n\mathrm{C}_0x^0 + _n\mathrm{C}_1x^1 + _n\mathrm{C}_2x^2 + \ldots + _n\mathrm{C}_nx^n \\ &=& \sum^{n}_{k=0} \binom{n}{k} x^k \end{eqnarray}\end{split}\]
二項係数は組み合わせ (combinations) の数でもある。
二項係数は「 \(n\) 個の中から \(k\) 個を選ぶ組み合わせの総数(場合の数)」「 \(n\) 個の要素を \(k\) 個と \((n-k)\) 個に分割する方法の総数 」と言い換えることができる。
\[\begin{eqnarray} _n\mathrm{C}_k = \frac{_n\mathrm{P}_k}{k!} = \frac{n!}{k!(n-k)!} \end{eqnarray}\]

性質¶

次の等式が成り立つ。

\[\begin{split}\begin{eqnarray} \binom{n}{k} &=& \binom{n}{n-k} \\ \end{eqnarray}\end{split}\]

また、次の漸化式が成り立つ。

\[\begin{split}\begin{eqnarray} \binom{n}{0} &=& 1 \\ \binom{n}{k} &=& \begin{cases} 0 & (k > n) \\ \binom{n-1}{k-1} + \binom{n-1}{k} & (k <= n) \\ \end{cases} \end{eqnarray}\end{split}\]

import numpy as np

重複組み合わせ¶

「 \(n\) 個の中から重複を許して \(r\) 個を選ぶ組み合わせ」を重複組み合わせという。

例えば4つの数の集合 \(\{1, 2, 3, 4\}\) について、重複を許して3個選ぶ組み合わせを考える。組み合わせには \(1234\) や \(1224\) などといった数列が含まれる。
これは、\(3\) 個の \(\bigcirc\) と \((4-1)\) 本の仕切りを並べる方法と読み替えることができる。
\[\begin{split}\begin{array}{cccc} \bigcirc & | & \bigcirc & | & \bigcirc & | & \bigcirc \\ \\ | & \bigcirc & \bigcirc & \bigcirc & | & | & \bigcirc \\ \end{array}\end{split}\]

上記の並びは次の数列に対応する。

\[\begin{split}\begin{array}{cc} 1 & 2 & 3 & 4 \\ \\ 2 & 2 & 2 & 4 \\ \end{array}\end{split}\]

「 \(n\) 個の中から重複を許して \(r\) 個を選ぶ組み合わせ」は「\(n\) 個の \(\bigcirc\) と \((r-1)\) 本の仕切りの並べ方」と言い換えることができ、場合の数は \(_{n+r-1}\mathrm{C}_r\) 通りとなる。

全探索¶

組み合わせの全探索¶

組み合わせ \(_n\mathrm{C}_r\) の全状態を列挙する方法を示す。

要素列 \(A = a_1 a_2 \ldots a_n\) を考える。
長さ \(r\) の組み合わせを列挙するには、順序関係が \(a_i \prec a_{i+1}\) となるように要素を \(r\) 個取り出して並べればよい。

言い換えると、ある操作で取り出した要素が \(a_i\) であるとき、次の操作では \((i+1)\) 番目以降の要素 \(a_{i+1}, \ldots, a_n\) のいずれかから選ぶことになる。

[3]:

def traverse_combinations(p, r):
    """組み合わせの列挙"""
    a = list(p)
    n = len(a)
    combs = list()
    state=[]

    def dfs(depth, l=0):
        if depth == r:
            yield tuple(state)
        else:
            for i in range(l, n):
                state.append(a[i])
                yield from dfs(depth+1, i+1)
                state.pop()

    yield from dfs(0)

[4]:

print(list(traverse_combinations(range(4), 3)))

[(0, 1, 2), (0, 1, 3), (0, 2, 3), (1, 2, 3)]

列挙だけであれば itertools.combinations のほうが高速。

重複組み合わせの全探索¶

重複組み合わせ \(_{n+r-1}\mathrm{C}_r\) の全状態を列挙する方法を示す。

組み合わせをベースに、順序関係が \(a_i \preceq a_{i+1}\) となるように並べればよい。

[6]:

def traverse_combinations_with_replacement(p, r):
    """重複組み合わせの列挙"""
    a = list(p)
    n = len(a)
    combs = list()
    state=[]

    def dfs(depth, l=0):
        if depth == r:
            yield tuple(state)
        else:
            for i in range(l, n):
                state.append(a[i])
                yield from dfs(depth+1, i)
                state.pop()

    yield from dfs(0)

print(list(traverse_combinations_with_replacement(range(3), 2)))

[(0, 0), (0, 1), (0, 2), (1, 1), (1, 2), (2, 2)]

こちらも列挙だけであれば itertools.combinations_with_replacement のほうが高速。

二項係数の計算¶

一般に二項係数は巨大な数となるため、断りがない限り \(\gcd(n, p)=1\) となる素数の法 \(p\) に関する合同を扱う。
(素数でなくとも、\(\gcd(n, m)=1\) となる整数の法 \(m\) に関して成り立つ)

方法1¶

\(n \leq 10^6\) 程度まで対応可能。

\[\binom{n}{k} = \frac{n!}{k!(n-k)!} = n!(k!)^{-1}((n-k)!)^{-1}\]
\(n\) 以下の全ての \(k\) に対して \(k!, (k!)^{-1}\) の2種類の値を事前計算しておくことで、クエリ毎に \(\mathcal{O}(1)\) で計算可能となる。
階乗 \(k!\) はforループで求めればよい。逆元計算は合同を参照。
[9]:

def binom_preprocess(n, MOD=10**9+7):
    f = [0 for i in range(n+1)]  # n!
    invf = [0 for i in range(n+1)]  # (n!)^-1
    f[0] = 1
    f[1] = 1
    invf[0] = 1
    invf[1] = 1
    for i in range(2, n+1):
        f[i] = f[i-1] * i % MOD
        invf[i] = pow(f[i], MOD-2, MOD)
    return f, invf

def binom(n, k, f, invf, MOD=10**9+7):
    if n < k or n < 0 or k < 0:
        return 0
    else:
        return (f[n] * invf[k] % MOD) * invf[n-k] % MOD
前処理の pow(f[i], p-2, p) を含むループはさらに計算量を落とすことができる。
\(a_k = (k!)^{-1}\) とおけば、漸化式 \(a_{k-1} = k \cdot a_k\) が成り立つので、 \(a_n = (n!)^{-1}\) だけ計算すれば他は漸化式から求まる。

計算量は事前計算に\(\mathcal{O}(n)\)、クエリ毎に\(\mathcal{O}(1)\)。

[10]:

def binom_preprocess(n, MOD=10**9+7):
    f = [0 for i in range(n+1)]  # n!
    invf = [0 for i in range(n+1)]  # (n!)^-1
    f[0] = 1
    f[1] = 1
    invf[0] = 1
    invf[1] = 1
    for i in range(2, n+1):
        f[i] = f[i-1] * i % MOD
    invf[n] = pow(f[n], MOD-2, MOD)
    for i in range(n, 2, -1):
        invf[i-1] = invf[i] * i % MOD
    return f, invf

def binom(n, k, f, invf, MOD=10**9+7):
    if n < k or n < 0 or k < 0:
        return 0
    else:
        return (f[n] * invf[k] % MOD) * invf[n-k] % MOD

[11]:

f, invf = binom_preprocess(10**6)
print(binom(10**6, 4, f, invf))

85121780

方法2¶

\(\min(k, n-k) \leq 10^6\) 程度まで対応可能。 \(n\) が大きくても対応できることが特徴。

\[\binom{n}{k} = \frac{n!}{k!(n-k)!} = \frac{n(n-1) \ldots (n-k+1)}{k(k-1) \ldots 1}\]
分母と分子を別々に計算し、それらの商をとる。
分母、分子はいずれも \(k\) 回のループで求まるので、全体の計算量はクエリ毎に \(\mathcal{O}(k)\)。
[12]:

def binom(n, k, MOD=10**9+7):
    k = min(k, n-k)
    a = 1
    for i in range(n-k+1, n+1):
        a = a * i % MOD
    b = 1
    for i in range(1, k+1):
        b = b * i % MOD
    return a * pow(b, MOD-2, MOD) % MOD

[13]:

print(binom(10**12, 10**6))

344082972

二項係数の和¶

\(n\) を固定したときの和¶

\[\sum^{n}_{k=0} \binom{n}{k} x^k = (1+x)^n\]

に対して \(x=1\) を代入すると

\[\begin{split}\begin{eqnarray} \sum^{n}_{k=0} \binom{n}{k} 1^k &=& (1+1)^n \\ \sum^{n}_{k=0} \binom{n}{k} &=& 2^n \\ \end{eqnarray}\end{split}\]

となる。つまり、\(n\) を固定したときの二項係数の和は \(2^n\) である。

\(k\) を固定した時の和¶

二項係数に関して \(k=1\) または \(k=2\) としたときの和は、和の公式や三角数と呼ばれる。
\(\gcd(n, m)=1\)となる法 \(m\) に関して計算量 \(\mathcal{O}(\log{m})\) で和が求まる。
\[\begin{split}\begin{eqnarray} 1 + 2 + 3 + 4 + \ldots + \binom{n}{1} &=& \frac{n(n+1)}{2} \\ 1 + 3 + 6 + 10 + \ldots + \binom{n}{2} &=& \frac{n(n+1)(n+2)}{6} \\ \end{eqnarray}\end{split}\]

任意の \(k\) に対しては、二項係数を計算してその和をとればよい。この場合は計算量が少し落ち、 \(\mathcal{O}(n\log{m})\) となる。
